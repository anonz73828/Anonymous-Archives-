Medusa

Medusa is a speedy, massively parallel, modular login brute-forcer designed for network services that support remote authentication. It enables concurrent testing against multiple hosts, users, or passwords, with support for protocols such as FTP, HTTP, SSH, Telnet, SNMP, and others. Medusa is particularly useful in ethical penetration testing to assess authentication mechanism strength and identify weak credentials.

Uses

Medusa is employed to perform brute-force and dictionary attacks on remote login services during authorized security assessments. Example: Testing FTP or web form authentication with credential lists to evaluate password policies and detect vulnerabilities to credential stuffing or weak authentication in controlled environments.

Installation in Termux

Medusa is not typically available as a pre-built package in Termux repositories and requires compilation from source. Install dependencies and build as follows:

pkg update && pkg upgrade -y
pkg install git build-essential libssh libssl-dev libpcre3-dev libidn libgtk2.0-dev libmysqlclient-dev libpq-dev libsvn-dev firebird-dev libmemcached-dev libgpg-error-dev libgcrypt-dev -y
git clone https://github.com/jmk-foofus/medusa.git
cd medusa
./configure
make
# Optional install: make install (may require root or manual path adjustment)
# Run: ./medusa -h

Common issues: Missing development libraries—install additional -dev packages as prompted; compilation failures—ensure all dependencies are met or use a proot-distro (e.g., Ubuntu) for a fuller build environment. To uninstall: Remove the cloned directory and related packages. Always verify source integrity before building.

Functions and Walkthroughs

Basic syntax: medusa -h target -u user -P passlist.txt -M module. List modules: medusa -d. Use host/user/pass lists: -H hosts.txt -U users.txt -P passwords.txt. Thread control: -T 10 (parallel tasks). Verbose: -v 4. Example FTP test: medusa -h 192.168.1.100 -u admin -P rockyou.txt -M ftp. Restrict usage to authorized targets only.

Detailed Tutorial: Dictionary Attack on FTP Service (Ethical Lab Scenario)

This tutorial demonstrates using Medusa to conduct a dictionary attack against an FTP service in a controlled lab environment (e.g., a virtual machine running an FTP server with a weak, known password for the user 'ftpuser'). This procedure is permitted only on systems you own or for which you have explicit written authorization. Unauthorized brute-force attempts constitute illegal access and may trigger defensive measures or legal consequences.

  1. Prepare the target: Set up a test FTP server (e.g., vsftpd on a VM at 192.168.1.100) with a weak password for 'ftpuser'. Ensure the service is accessible from your Termux device.
  2. Verify connectivity: Test basic FTP reachability with ftp 192.168.1.100 (do not proceed without permission).
  3. Run Medusa dictionary attack: Execute medusa -h 192.168.1.100 -u ftpuser -P /path/to/rockyou.txt -M ftp -T 5 -v 4. Here, -u specifies a single username, -P the password list, -M selects the FTP module, -T limits concurrent threads to 5 (adjust for performance), and -v 4 enables detailed verbose output.
  4. Monitor progress: Medusa displays ongoing attempts, successful logins (e.g., ACCOUNT FOUND: [ftp] Host: 192.168.1.100 User: ftpuser Password: password123), and summary statistics upon completion or interruption.
  5. Analyze results: If a valid credential is found, Medusa reports it clearly. Note any rate-limiting, account lockouts, or connection refusals triggered on the target.
  6. Alternative with combined lists: For multiple users and hosts, use medusa -H hosts.txt -U users.txt -P passwords.txt -M ftp -T 8.
  7. Clean up and report: Interrupt with Ctrl+C if necessary. Document findings responsibly, recommending multi-factor authentication, strong password policies, or monitoring for brute-force attempts.

Expected results: Identification of weak credentials if present in the wordlist; otherwise, exhaustion of candidates with no valid login reported. Common issues: Protocol-specific errors—verify module options with medusa -M ftp -q; high thread counts may cause detection—reduce -T value accordingly.
