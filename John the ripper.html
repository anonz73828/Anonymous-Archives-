John the Ripper

John the Ripper is an open-source password security auditing and recovery tool developed by Openwall. It supports a wide range of hash types and ciphers, with the "jumbo" community edition offering hundreds of additional formats and optimizations for CPU and GPU cracking. It excels in offline password recovery from sources such as /etc/shadow files, Windows SAM dumps, password-protected archives, and more.

Uses

John the Ripper is used for ethical password strength evaluation, detecting weak credentials in system dumps or files during authorized penetration testing and security audits. Example: Cracking unsalted MD5 hashes or testing Unix shadow passwords with wordlists to assess policy compliance and recommend improvements.

Installation in Termux

John the Ripper (jumbo edition recommended) requires compilation from source in Termux, as pre-built packages are not standard. Use the following steps:

pkg update && pkg upgrade -y
pkg install git build-essential clang make libssl-dev zlib-dev -y
git clone https://github.com/openwall/john.git
cd john/src
./configure && make -s clean && make -sj4
cd ../run
# Test: ./john --help
# Alternative one-liner installer script (verify source first):
pkg install wget -y && wget https://raw.githubusercontent.com/shahriarkabir-sagor/Termux-JohnTheRipper-Installer/master/jtr-installer.sh && chmod +x jtr-installer.sh && ./jtr-installer.sh

Common issues: Compilation errors—install missing devel packages; limited performance—Termux uses CPU only (no full GPU support). To uninstall: Remove the cloned directory. For optimal speed, use a desktop environment with GPU acceleration.

Functions and Walkthroughs

Basic usage: ./john hashes.txt (auto-detects mode/format). Wordlist mode: ./john --wordlist=rockyou.txt hashes.txt. Show cracked passwords: ./john --show hashes.txt. Incremental mode: ./john --incremental hashes.txt. Format specification: ./john --format=raw-md5 hashes.txt. Always apply to authorized data only.

Detailed Tutorial: Wordlist Attack on MD5 Hashes (Ethical Lab Scenario)

This tutorial demonstrates cracking unsalted MD5 password hashes using a dictionary attack in a controlled lab (e.g., self-generated test hashes). Perform this exclusively on data you own or have explicit written permission to analyze. Unauthorized password recovery is illegal.

  1. Prepare test hashes: Create a file (e.g., hashes.txt) with MD5 hashes. Example content (one per line): 5f4dcc3b5aa765d61d8327deb882cf99 (MD5 of "password"). Use tools like unshadow for real /etc/passwd + /etc/shadow combos: unshadow passwd shadow > hashes.txt.
  2. Navigate to run directory: cd ~/john/run (or wherever compiled).
  3. Run wordlist attack: ./john --wordlist=/path/to/rockyou.txt --format=raw-md5 hashes.txt. John auto-detects if format omitted in many cases.
  4. Monitor progress: John displays status (guesses per second, ETA). Press any key for interim status or Ctrl+C to pause/resume.
  5. View cracked results: ./john --show --format=raw-md5 hashes.txt. Output reveals cracked pairs (e.g., ?:password for the example hash).
  6. Resume session: Re-run the same command—John resumes from potfile (cracked hashes saved in ~/.john/john.pot).
  7. Advanced: Add rules for mangling: ./john --wordlist=rockyou.txt --rules hashes.txt. For incremental brute-force: ./john --incremental=All hashes.txt.
  8. Clean up and report: Securely delete sensitive files post-test. Document weak passwords and advocate for stronger hashing (e.g., bcrypt, Argon2) and policies.

Expected results: Rapid cracking of common/weak passwords if in the wordlist; longer times for complex ones. Common issues: Format mismatch—specify --format; slow on mobile CPU—limit scope or use desktop. Compare to Hashcat for GPU-accelerated scenarios (John excels in CPU and certain formats).
