NetworkMiner

NetworkMiner is an open-source network forensic analysis tool (NFAT) developed by NETRESEC. It passively analyzes pcap or pcapng files (or live network interfaces) to extract and reassemble application-layer artifacts such as transmitted files, credentials, certificates, images, emails, and keystrokes without sending any packets or altering traffic. NetworkMiner presents extracted data in a structured, tabbed interface organized by host, session, file, credential, and parameter views, making it exceptionally useful for incident response, malware analysis, post-exploitation forensics, and validating data exfiltration or credential theft during authorized security assessments.

Uses

NetworkMiner is employed in ethical forensic investigations to extract files, credentials, and other artifacts from captured traffic, reconstruct file transfers (HTTP, SMB, SMTP, FTP, TFTP), identify malware download or C2 channels, recover session cookies, and map communication patterns without active probing. Example: Analyzing a pcap file from a compromised system in a controlled lab to recover exfiltrated documents, stolen credentials, or dropped malware payloads during an authorized incident response or red team debrief.

Installation in Termux

NetworkMiner is a .NET Windows application requiring the Mono runtime for Linux/Android execution. Full GUI operation in Termux requires proot-distro (Ubuntu) with X11/VNC forwarding. Steps:

pkg update && pkg upgrade -y
pkg install proot-distro -y
proot-distro install ubuntu
proot-distro login ubuntu
# Inside Ubuntu chroot:
apt update && apt upgrade -y
apt install mono-complete wget unzip -y
# Download latest release (check https://www.netresec.com/?page=NetworkMiner for current version)
wget https://www.netresec.com/?download=NetworkMiner -O NetworkMiner.zip
unzip NetworkMiner.zip -d NetworkMiner
cd NetworkMiner
# Run GUI (requires X11/VNC): mono NetworkMiner.exe
# For CLI extraction (limited): mono NetworkMiner.exe -r capture.pcap

Common issues: GUI not displaying—configure VNC/X11 forwarding; Mono version mismatch—ensure mono-complete is installed; large pcap files—mobile storage and memory constraints. To uninstall: Remove the proot-distro or extracted directory. Optimal analysis (especially large captures) is best performed on a desktop Linux/Windows system.

Functions and Walkthroughs

Launch GUI: mono NetworkMiner.exe. Open pcap: File → Open → Select capture file. View tabs: Hosts (IP/MAC/hostname), Files (carved/reassembled files), Credentials (username/password pairs), Sessions (TCP/UDP flows), Parameters (HTTP POST/GET data), DNS (queries/responses), Keywords (custom search hits). Export artifacts: Right-click files/credentials → Save As. Live capture: Capture → Start (requires libpcap/WinPcap). Always analyze traffic from authorized sources only.

Detailed Tutorial: Passive Artifact Extraction from pcap File (Ethical Lab Scenario)

This tutorial demonstrates using NetworkMiner to passively extract files, credentials, and other artifacts from a pre-captured pcap file in a controlled lab environment (e.g., traffic generated from test systems you own). This procedure is permitted only on captures from networks and systems you control or for which you have explicit written authorization. Unauthorized analysis of network traffic is illegal in many jurisdictions.

  1. Prepare test capture: Generate sample traffic in a lab (e.g., browse HTTP sites, upload/download files via FTP/SMB, send emails via SMTP, or simulate credential entry on an insecure web form). Capture the traffic using tcpdump, Wireshark, or Bettercap and save as capture.pcap.
  2. Launch NetworkMiner: In the Ubuntu chroot (or desktop), run mono NetworkMiner.exe. Ensure X11/VNC is configured if using GUI in Termux emulation.
  3. Open the pcap file: File → Open → Navigate to capture.pcap. NetworkMiner automatically parses the file and populates all tabs (may take several seconds to minutes depending on capture size).
  4. Review Hosts tab: Examine the list of observed IP addresses, hostnames, MAC addresses, operating systems (via fingerprinting), and session counts. Double-click any host to see detailed communication patterns.
  5. Examine Files tab: View reassembled and carved files (e.g., downloaded images, uploaded documents, malware binaries). Right-click entries → Save As to extract files to disk. Check MD5/SHA hashes for known-good/malicious comparison.
  6. Inspect Credentials tab: Look for extracted username/password pairs from cleartext protocols (HTTP Basic, FTP, Telnet, SMTP AUTH, etc.) or form POST data. NetworkMiner highlights sensitive fields automatically.
  7. Check other tabs: Parameters tab shows HTTP form data and cookies; Sessions tab lists TCP/UDP flows with duration and byte counts; Keywords tab allows custom string searches across the capture.
  8. Export and document: Use File → Export Objects → All Files/Hosts/Credentials to save artifacts. Generate a report (limited in free version) or manually document findings (e.g., exfiltrated files, leaked credentials, suspicious C2 domains). Recommend defenses: enforce HTTPS everywhere (HSTS), disable insecure protocols, implement network segmentation, DLP, and continuous monitoring.

Expected results: Structured reconstruction of transmitted files, credentials, certificates, and session details directly from passive pcap analysis, often revealing data leakage or malicious activity not easily visible in traditional packet viewers. Common issues: Incomplete reassembly—large/fragmented transfers may require manual Wireshark follow-up; encrypted traffic—NetworkMiner cannot extract inside TLS unless keys are available; large captures—memory constraints on mobile emulation.
