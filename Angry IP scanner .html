Angry IP Scanner

Angry IP Scanner is a free, open-source, cross-platform network scanner designed for speed and simplicity. It scans IP ranges or individual addresses, pings hosts to determine live status, resolves hostnames, detects MAC addresses (via ARP), and scans for open ports. Additional fetchers retrieve NetBIOS names, shared folders, web server titles, and other information. It is widely used by network administrators for inventory, troubleshooting, and ethical reconnaissance on authorized networks.

Uses

Angry IP Scanner is employed for discovering live hosts, mapping network topology, identifying open ports, and gathering basic device information during authorized security audits or network management tasks. Example: Scanning a local subnet (e.g., 192.168.1.0/24) to inventory connected devices, verify IP usage, or detect unauthorized hosts in a controlled environment.

Installation in Termux

Angry IP Scanner is a Java-based GUI application (requires Java Runtime Environment 11+). It is not natively supported in Termux due to Android's GUI limitations. Use a proot-distro with X11/VNC for emulation, or run on a desktop OS:

# Recommended: Download from official site on desktop
# Visit https://angryip.org/download/ and select your platform (Windows, Mac, Linux DEB/RPM, or portable JAR)
# Requires Java 11+ (install via package manager or bundled versions)
# For Termux/Android (limited GUI via VNC/X11):
pkg install proot-distro -y
proot-distro install ubuntu
proot-distro login ubuntu
# Inside Ubuntu chroot:
apt update && apt upgrade -y
apt install default-jre wget -y
# Download latest JAR or DEB (adjust version):
wget https://github.com/angryip/ipscan/releases/download/3.9.3/ipscan-3.9.3.jar
# Run: java -jar ipscan-3.9.3.jar
# For GUI: Set up VNC/X11 forwarding (requires external VNC viewer app)

Common issues: GUI not rendering—configure VNC/X11 properly; performance lag—mobile emulation is slower than native desktop. To uninstall: Remove the proot-distro or JAR file. Full functionality and speed are best achieved on a desktop environment.

Functions and Walkthroughs

Basic scan: Enter IP range (e.g., 192.168.1.1 - 192.168.1.254) → Fetchers → Start. View results in columns (IP, Ping, Hostname, Ports, etc.). Export: File → Export As (CSV, TXT, XML). Preferences: Tools → Preferences for timeouts, threads, and fetchers. Command-line: java -jar ipscan.jar -f:range 192.168.1.1 192.168.1.254 -o results.txt. Always scan networks you own or have explicit written permission to access.

Detailed Tutorial: Basic Network Host Discovery (Ethical Lab Scenario)

This tutorial demonstrates scanning a local subnet to identify live hosts and basic details using Angry IP Scanner in a controlled lab or home network you own. Perform scans exclusively on networks with explicit authorization. Unauthorized scanning of external or third-party networks may violate laws or terms of service.

  1. Launch Angry IP Scanner: Open the application (via desktop shortcut, JAR execution, or emulated GUI in Termux).
  2. Configure scan range: In the "IP Range" field, enter your local subnet (e.g., 192.168.1.1 - 192.168.1.254 or CIDR 192.168.1.0/24). Use "Pick from network interfaces" for auto-detection of your current LAN.
  3. Select fetchers: Click the "Fetchers" menu (or right-click column headers) → Enable desired columns (e.g., Hostname, MAC Address, Ping, Open Ports). Add port scanners via Fetchers → Add → Ports (e.g., check common ports 80, 443, 22, 445).
  4. Start the scan: Click the "Start" button (play icon). The tool pings each address rapidly and populates results in real time (live hosts turn green; dead ones gray).
  5. Monitor and filter: Watch progress in the status bar. Use filters (e.g., show only alive hosts) or sort columns. Pause/resume as needed.
  6. Analyze results: Review columns for IP, status, hostname (via DNS/NetBIOS), MAC vendor, open ports, and additional fetched data (e.g., web titles if port 80/443 open).
  7. Export data: File → Export As → Choose format (CSV recommended) → Save file for documentation or further analysis.
  8. Clean up and report: Close the application. Document findings responsibly (e.g., inventory devices, note anomalies like unexpected hosts). Recommend network segmentation or monitoring for unauthorized devices.

Expected results: A table of live hosts with details (e.g., router at .1, printers, phones, computers). Common issues: Slow scans—increase threads in Preferences; no MAC addresses—requires local subnet and ARP; firewall blocks—some hosts appear dead. For advanced usage, explore custom fetchers or command-line automation.
