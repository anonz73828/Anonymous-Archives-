DarkComet (Historical Reference Only)

DarkComet was a Remote Access Tool (RAT) developed by Jean-Pierre Lesueur (known online as DarkCoderSc) between approximately 2008 and 2012. Initially marketed as a legitimate remote administration utility, it gained notoriety due to its extensive use by malicious actors for unauthorized access, surveillance, data exfiltration, and persistence on victim systems. The project was officially discontinued by its author in 2012 following public exposure of its abuse in high-profile cyber-espionage and cybercrime campaigns. This page exists solely for educational and defensive purposes: to aid security professionals, malware analysts, and defenders in understanding RAT functionality, persistence mechanisms, detection signatures, and mitigation strategies.

Important Legal and Ethical Notice: DarkComet is classified as malware by virtually all reputable antivirus vendors and security organizations. Possession, distribution, or deployment of DarkComet (or its variants) without explicit, written authorization from all affected system owners is illegal in most jurisdictions and constitutes unauthorized access, computer fraud, or espionage. This resource provides no downloads, installation instructions, source code, or operational guidance. Any attempt to acquire or use DarkComet outside of legitimate malware research in isolated, air-gapped laboratory environments may result in criminal liability.

Historical Capabilities (Defensive Awareness)

DarkComet featured a modular client-server architecture with the following documented capabilities (observed in samples from 2010â€“2012 era):

  • Remote desktop viewing and control
  • Keylogging and clipboard capture
  • File manager (upload/download, execute, delete)
  • Password recovery from browsers and applications
  • Webcam and microphone access
  • Registry and process manipulation
  • DDNS integration and port forwarding
  • Anti-VM/sandbox detection and persistence mechanisms (startup entries, scheduled tasks, service installation)
  • Customizable server builder with basic obfuscation

These features made it attractive to threat actors but also left numerous detectable artifacts (registry keys, file paths, network beacons, mutexes, and specific strings).

Detection and Mitigation (Defensive Focus)

Modern endpoint detection tools identify DarkComet through:

  • Signature-based detection (known hashes, YARA rules)
  • Behavioral indicators (unusual process injection, screen capture APIs, keylogging hooks)
  • Network IOCs (default ports 160, 81, 8080; C2 domains; specific HTTP user-agents or packet patterns)
  • File artifacts (e.g., %AppData%\DarkComet, specific .ini or .key files)

Recommended defenses include application whitelisting, least-privilege accounts, network segmentation, EDR/XDR monitoring, and regular patching. Legacy systems vulnerable to DarkComet-era exploits should be decommissioned or isolated.

Conclusion: DarkComet serves as an important case study in the evolution of RATs, the risks of dual-use tools, and the importance of proactive defense. Modern equivalents (both legitimate remote administration software and malicious RATs) continue to exist; understanding DarkComet helps defenders recognize similar tactics, techniques, and procedures (TTPs) today.
