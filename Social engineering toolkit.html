Social-Engineer Toolkit (SET)

The Social-Engineer Toolkit (SET) is an open-source penetration testing framework developed by TrustedSec specifically designed to automate and streamline social engineering attacks. It enables security professionals to create realistic phishing campaigns, harvest credentials, deliver malicious payloads, exploit client-side vulnerabilities, and simulate advanced social engineering scenarios. SET integrates tightly with Metasploit and supports a wide range of attack vectors, including website cloning, Java applet attacks, tabnabbing, QR code generation, and SMS-based phishing. The Community Edition is free and widely used in authorized red team exercises to evaluate human susceptibility and organizational security awareness.

Uses

SET is employed in ethical penetration testing to simulate real-world social engineering attacks, assess user awareness, test defensive controls (e.g., email filtering, endpoint protection), and demonstrate the human element in security breaches. Example: Cloning a legitimate login page to harvest credentials from test users in an authorized awareness campaign, or delivering a malicious payload disguised as a software update to evaluate endpoint detection capabilities in a controlled lab environment.

Installation in Termux

SET is Python-based and installs easily in Termux. It requires git, Python 3, and Metasploit (optional but recommended for payload delivery). Installation steps:

pkg update && pkg upgrade -y
pkg install git python python-pip -y
git clone https://github.com/trustedsec/social-engineer-toolkit.git setoolkit
cd setoolkit
pip install -r requirements.txt
python setup.py install   # or just run directly: python setoolkit
# Launch: python setoolkit
# First run will prompt to agree to terms and may install additional dependencies

Common issues: Missing dependencies—rerun pip install -r requirements.txt; Metasploit integration—install msfconsole separately if needed; port conflicts—ensure no other service uses ports 80/443. To uninstall: Remove the setoolkit directory. Full functionality (especially payload delivery) is enhanced when paired with Metasploit running in the same environment.

Functions and Walkthroughs

Launch SET: python setoolkit. Main menu options: 1) Social-Engineering Attacks, 2) Penetration Testing (Fast-Track), 3) Third Party Modules, etc. Key attack vectors: Website Attack Vectors (credential harvester, tabnabbing, Java applet), Spear-Phishing Attack Vectors, Infectious Media Generator, QRCode Generator Attack Vector. Select options numerically; follow prompts to configure payloads, listener IPs/ports, and templates. Always test in isolated lab networks with explicit permission.

Detailed Tutorial: Credential Harvesting via Website Cloning (Ethical Lab Scenario)

This tutorial demonstrates using SET to clone a legitimate login page and harvest credentials in a controlled lab environment (e.g., testing your own awareness training program or a deliberately vulnerable test setup you own). This procedure is permitted only in authorized environments with explicit written permission from all participants and system owners. Unauthorized phishing or credential harvesting is illegal and unethical.

  1. Launch SET: Run python setoolkit in Termux (or your environment). Accept the terms if prompted.
  2. Select attack type: Choose option 1) Social-Engineering Attacks → 2) Website Attack Vectors → 3) Credential Harvester Attack Method → 2) Site Cloner.
  3. Configure listener: Enter your IP address for the reverse handler (e.g., your Termux device IP: 192.168.1.50). SET will start a local web server on port 80.
  4. Specify target URL: Enter the legitimate login page URL to clone (e.g., https://example.com/login or a test page like http://localhost/dvwa/login.php). SET will clone the site and modify form action to post credentials back to your server.
  5. Optional: Select templates (e.g., Gmail, Office 365, custom) or use the cloned site directly. SET starts Apache/HTTP server and displays "Credential Harvester is running on port 80".
  6. Distribute phishing link (lab only): Send the generated URL (http://your-ip) to test users via email, QR code, or other means within the authorized scope. Monitor SET console for captured credentials.
  7. Observe results: When a test user submits credentials on the cloned page, SET displays them in real time (e.g., username: testuser, password: Password123!). Credentials are also logged to /var/www/html/harvester_* or similar.
  8. Stop and clean up: Press Ctrl+C in SET to terminate. Document captured data responsibly (for training purposes only). Recommend defenses: email filtering, user awareness training, MFA enforcement, and browser security indicators.

Expected results: Successful capture of test credentials entered on the cloned page, demonstrating the effectiveness of phishing simulation. Common issues: Port 80/443 blocked—use a different port or run SET with elevated privileges; HTTPS cloning—use a real domain with SSL for realism (requires certificate); firewall/IDS detection—modern defenses may flag suspicious behavior.

For authoritative reference, consult the official resources: Social-Engineer Toolkit GitHub Repository, TrustedSec SET Documentation, and Social-Engineer.org Framework (complementary reading).
