Unicornscan

Unicornscan is an asynchronous, stateless TCP and UDP port scanner designed for high speed and low resource consumption. Unlike traditional synchronous scanners, it sends packets without waiting for responses before sending the next, enabling extremely fast scans of large networks. It includes built-in OS fingerprinting, banner grabbing, custom payload support, and detailed response analysis. Unicornscan is particularly valuable in ethical penetration testing and network reconnaissance when rapid, stealthy, or high-volume scanning is required.

Uses

Unicornscan is utilized for fast discovery of live hosts, open ports, service banners, and operating system identification during authorized security assessments. Example: Performing a rapid scan of a large subnet to identify potentially vulnerable services or map network topology in a controlled lab environment, with minimal detection footprint compared to slower tools like Nmap in certain scenarios.

Installation in Termux

Unicornscan is not available as a pre-built package in Termux repositories and must be compiled from source. Raw socket access typically requires root privileges (use Termux with tsu/root or a rooted device). Installation steps:

pkg update && pkg upgrade -y
pkg install git build-essential autoconf automake libtool libpcap libdnet libpcre bison flex zlib-dev -y
git clone https://github.com/dvastbinder/unicornscan.git
cd unicornscan
./bootstrap.sh
./configure --prefix=/data/data/com.termux/files/usr --with-pcap-includes=/data/data/com.termux/files/usr/include --with-pcap-libraries=/data/data/com.termux/files/usr/lib
make
make install
# Test: unicornscan -h
# Note: If configure fails on libdnet, install libdumbnet-dev or build libdnet separately

Common issues: Permission denied on raw sockets—run with root (tsu or su); missing dependencies—install additional -dev packages; compilation errors—verify autotools and bison/flex versions. To uninstall: Remove the cloned directory and run make uninstall if installed. Full functionality often requires a rooted device or Kali NetHunter for raw packet capabilities.

Functions and Walkthroughs

Basic TCP scan: unicornscan 192.168.1.0/24:a -r 1000 (scans all ports asynchronously at 1000 packets/sec). UDP scan: unicornscan -mU 192.168.1.100. OS fingerprint: unicornscan -v -mT 192.168.1.100 (verbose TCP with fingerprint). Banner grab: unicornscan -mT -b 192.168.1.100:80. Custom rate/payload: unicornscan -r 5000 -p 1-65535 -i eth0 192.168.1.100. Always scan networks you own or have explicit written permission to access.

Detailed Tutorial: High-Speed Port Scan with OS Fingerprinting (Ethical Lab Scenario)

This tutorial demonstrates using Unicornscan to perform a fast, asynchronous TCP port scan with OS fingerprinting and banner grabbing on a controlled lab network (e.g., a local subnet with test VMs you own). This procedure is permitted only on networks and systems you control or for which you have explicit written authorization. Unauthorized scanning is illegal and may be detected as hostile activity.

  1. Prepare the environment: Ensure Unicornscan is compiled and installed. Identify your network interface (e.g., wlan0) with ifconfig or ip link. Confirm you are on the target subnet or have routing permission.
  2. Run a basic high-speed scan: Execute unicornscan -v -i wlan0 192.168.1.0/24:a -r 2000. The -v enables verbose output, -i specifies interface, :a scans all ports, -r sets packet rate (adjust lower if network saturates).
  3. Monitor output: Unicornscan displays real-time results including open ports (e.g., 80 open tcp http), response times, and TTL values useful for OS inference. Press Ctrl+C to stop early.
  4. Targeted scan with fingerprinting: Scan a specific host unicornscan -v -mT -b 192.168.1.100:1-1000,8080,8443. -mT forces TCP module, -b enables banner grabbing on detected open ports.
  5. Analyze results: Look for lines such as: 192.168.1.100:80:tcp open http - Apache/2.4.41 (Unix) - banner TTL 64 → likely Linux; TTL 128 → Windows; window size and options aid fingerprinting
  6. Advanced options: Use custom payloads for evasion unicornscan -mT -p payload=GET / HTTP/1.0\r\n\r\n 192.168.1.100:80, or UDP scan unicornscan -mU -r 500 192.168.1.100. Save output: Redirect with > results.txt.
  7. Compare and document: Contrast speed/results with Nmap (Unicornscan is typically faster for large ranges but less reliable for filtered ports). Record findings responsibly (live hosts, open services, banners) and recommend firewall hardening, service hardening, or intrusion detection.

Expected results: Extremely fast completion of large-range scans with detailed port status, banners, and basic OS inference. Common issues: Packet loss at high rates—reduce -r; root required—use tsu/su; no banner—target closed ports or filtered responses. Unicornscan complements rather than replaces Nmap in most modern workflows.
