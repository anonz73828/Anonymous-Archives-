Wireshark

Wireshark is the world's leading open-source network protocol analyzer. It captures live network traffic, displays packets in detailed human-readable form, and supports deep inspection of hundreds of protocols for troubleshooting, analysis, and security auditing.

Uses

Wireshark is used for packet capture, protocol dissection, traffic analysis, identifying anomalies (e.g., unusual patterns indicating attacks), and verifying network behavior. Example: Capturing HTTP traffic to inspect unencrypted data or detecting port scans in ethical assessments.

Installation in Termux

Wireshark requires a graphical environment in Termux (no native CLI-only capture without root). Use X11/VNC setup:

pkg update && pkg upgrade -y
pkg install x11-repo -y
pkg install wireshark wireshark-gtk xterm tigervnc -y
vncserver :1 -geometry 1280x720  # Set password when prompted
# Then use a VNC viewer app to connect to localhost:5901

Launch Wireshark via VNC session: wireshark &. Common issues: GUI not displaying—ensure VNC is connected; permissions—run termux-setup-storage. For basic CLI capture (limited), use tcpdump instead (pkg install tcpdump).

Functions and Walkthroughs

Select interface and start capture; apply display filters (e.g., http); follow streams (right-click packet > Follow > TCP Stream). Export objects or save captures (.pcapng). Ethical note: Capture only on networks you own or have authorization for.

Detailed Tutorial: Basic Packet Capture and Analysis (Ethical Lab Scenario)

This tutorial covers capturing and analyzing traffic in a controlled lab (e.g., local VM network or test setup). Perform captures only with explicit permission on authorized networks. Unauthorized interception may violate laws.

  1. Launch Wireshark via VNC/X11 session in Termux.
  2. Select capture interface: Choose the active network (e.g., wlan0 or eth0) from the welcome screen and click Start (blue shark fin icon).
  3. Generate traffic: Open a browser or tool to create packets (e.g., visit HTTP sites, ping hosts, or simulate protocols).
  4. Stop capture: Click the red square icon after sufficient packets are collected.
  5. Apply display filter: In the filter bar, enter http and press Enter to show only HTTP traffic; or tcp.port == 80 for port-specific.
  6. Inspect details: Select a packet in the list pane; examine protocol layers in the packet details pane (expand HTTP > Line-based text data for payloads).
  7. Follow stream: Right-click an HTTP packet > Follow > HTTP Stream to reconstruct requests/responses (e.g., view form data or credentials in cleartext—highlight risks).
  8. Analyze anomalies: Look for patterns like repeated SYN packets (port scans) or unusual ICMP (recon). Use Statistics > Conversations for summaries.
  9. Save capture: File > Save As (.pcapng) for later review or reporting.

Expected outcome: Visible packet breakdown revealing protocol details, potential cleartext data, or attack indicators. Common issues: No packets—ensure correct interface and promiscuous mode if available. For advanced filters: ip.src == 192.168.1.100 && http.request.
