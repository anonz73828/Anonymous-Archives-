RainbowCrack

RainbowCrack is an implementation of the time-memory trade-off technique for reversing cryptographic hash functions, originally developed by Philippe Oechslin. It enables extremely fast password recovery from captured password hashes when a precomputed rainbow table exists for the target hash algorithm, character set, and password length. Rainbow tables trade significant storage requirements for near-instantaneous lookup time, making RainbowCrack highly effective for offline auditing of unsalted or weakly protected hashes (e.g., LM, NTLM, MD5, SHA-1) during authorized security assessments.

Uses

RainbowCrack is employed in ethical password auditing to recover plaintext from password hashes obtained from system dumps, SAM files, NTDS.dit extracts, or other sources during authorized penetration testing. Example: Quickly cracking NTLM hashes from a captured Windows domain credential dump using precomputed rainbow tables to evaluate password strength and enforce stronger policies.

Installation in Termux

RainbowCrack is not packaged in Termux repositories and requires compilation from source. Performance is CPU-limited on mobile devices; GPU acceleration is not supported in Termux. Installation steps:

pkg update && pkg upgrade -y
pkg install git build-essential clang make -y
git clone https://github.com/RainbowCrack/rainbowcrack.git
cd rainbowcrack
# Build (may need adjustments for Termux paths)
make
# Test: ./rtgen -h
# Or download pre-built binaries for Linux from project site (if compatible)
# wget https://project-rainbowcrack.com/rainbowcrack-1.8-linux64.zip
# unzip rainbowcrack-1.8-linux64.zip
# cd rainbowcrack-1.8-linux64

Common issues: Compilation errors—install additional build tools or use proot-distro Ubuntu chroot; no GPU—Termux lacks CUDA/OpenCL support; large tables—storage constraints on mobile. To uninstall: Remove the cloned or extracted directory. Optimal performance requires a desktop with significant storage and CPU/GPU resources for table generation and lookup.

Functions and Walkthroughs

Generate table: rtgen hash_algorithm charset plaintext_len_min plaintext_len_max table_index chain_len chain_num -o table.rt. Sort table: rtsort table.rt. Crack hash: rcrack *.rt -h hash_value or rcrack *.rt -l hashlist.txt. List supported hashes: rcrack -h. Use precomputed tables from public repositories (ethical use only on authorized hashes). Operate exclusively on data you own or have explicit written permission to analyze.

Detailed Tutorial: Cracking NTLM Hashes with Precomputed Rainbow Tables (Ethical Lab Scenario)

This tutorial demonstrates using RainbowCrack with precomputed NTLM rainbow tables to recover plaintext passwords from captured NTLM hashes in a controlled lab environment (e.g., self-generated hashes from a test Windows system). This procedure is permitted only on data you own or for which you have explicit written authorization. Unauthorized password recovery is illegal.

  1. Obtain or generate test hashes: Create a test Windows account with a known weak password (e.g., "password123"). Extract NTLM hash using tools such as pwdump, secretsdump (Impacket), or Mimikatz in a lab VM (e.g., output format: user:rid:lmhash:nthash:::). Example NTLM hash: 8743b52063cd84097a65d1633f5c74f5 (NT hash of "password123"). Save to hashes.txt.
  2. Acquire rainbow tables: Download precomputed NTLM tables for common character sets and lengths from public repositories (e.g., freerainbowtables project archives or trusted mirrors). For this example, assume you have alpha-numeric 1–8 character tables (e.g., ntlm_alpha-numeric#1-7_0_2400x10000000_*.rt). Place tables in the working directory. Note: Generating your own tables requires significant time and storage (hundreds of GB for comprehensive coverage).
  3. Sort tables (if not already sorted): Run rtsort ntlm_alpha-numeric#*.rt for each table file. Sorted tables are required for lookup.
  4. Perform lookup: Execute rcrack *.rt -l hashes.txt or rcrack ntlm_*.rt -h 8743b52063cd84097a65d1633f5c74f5. RainbowCrack searches the chains in the tables and outputs plaintext if a match is found (e.g., hex:8743b52063cd84097a65d1633f5c74f5 -> plaintext:password123).
  5. Analyze results: If the password exists in the table coverage, recovery is near-instantaneous. If not found, the tool reports "not found" — indicating the password is outside the table parameters or uses a different charset/length.
  6. Compare with online cracking: For educational purposes, contrast RainbowCrack speed with brute-force tools (e.g., Hashcat) on the same hash. RainbowCrack excels when tables exist; otherwise, it offers no advantage.
  7. Document and remediate: Record findings responsibly. Recommend moving to modern hashing (bcrypt, Argon2, PBKDF2 with high iterations), enforcing complex password policies, and implementing multi-factor authentication to mitigate offline attacks.

Expected results: Near-instant recovery of passwords covered by the tables; "not found" for uncovered passwords. Common issues: Missing tables—ensure correct charset/length coverage; large table sets—require substantial disk space; unsorted tables—always run rtsort first. RainbowCrack remains most useful for legacy unsalted hashes (LM/NTLM); modern salted algorithms render it ineffective without enormous custom tables.

For authoritative reference and downloads (tables and binaries), consult: RainbowCrack Official Project Site, Free Rainbow Tables, and the included documentation (README, usage.txt) in the source distribution.
